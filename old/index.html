Travelling Salesman Solution via Simulated Annealing

NOTE: You must have a Java enabled browser e.g. Netscape 32-bit or Internet Explorer to view the above applet. If your browser does not support Java, or you have enable Java applets turned off in your preferences, there will be a static graphic saying "you need java", or nothing between the two horizontal separator lines above.

This demo demonstrates a possible solution to the classic Traveling Salesman Problem. It attempts to find the shortest round-trip connecting the set of cities that a salesman must visit. It places about a hundred cities at random points in the applet's area, and then uses the simulated annealing algorithm to estimate the shortest path. The status bar of your browser should be continuously updated with the current total path length as well as the Temperature. More on the temperature later. When the applet has finished, it will display the text "DONE:" in the status bar followed by the final path length. Click on the [Refresh] or [Reload] button in your browser to start the applet over. The applet slows down the action by including a delay between steps.

For more information about the Travelling Salesman, and why his problem is important to computer science, visit Travelling Salesmen Explained.
